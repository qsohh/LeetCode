## Complex Number Multiplication

好在正则表达式运行够快 ：

```python
class Solution:
    def complexNumberMultiply(self, num1: str, num2: str) -> str:
        r1 = int(re.search(r'.+(?=\+)', num1).group())
        i1 = int(re.search(r'(?<=\+).+(?=i)', num1).group())
        r2 = int(re.search(r'.+(?=\+)', num2).group())
        i2 = int(re.search(r'(?<=\+).+(?=i)', num2).group())
        return str(r1*r2-i1*i2) + '+' + str(r1*i2+r2*i1) + 'i'
```

但我其实不太明白为甚么直接找到```'+'```的位置然后分开会比这个慢 ：

```python
class Solution:
    def complexNumberMultiply(self, num1: str, num2: str) -> str:
        ind1, ind2 = num1.find('+'), num2.find('+')
        r1, i1 = int(num1[:ind1]), int(num1[ind1+1:-1])
        r2, i2 = int(num2[:ind2]), int(num2[ind2+1:-1])
        return str(r1*r2-i1*i2) + '+' + str(r1*i2+r2*i1) + 'i'
```
